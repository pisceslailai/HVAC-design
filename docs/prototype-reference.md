# 原型参考说明

原型用于固定信息架构、密度和主要交互，不代表生产代码或最终品牌视觉。可点击版本位于 `prototype/`，主截图位于 `output/playwright/`。

## 设计系统

- 风格：工业计算工作台，强调结构、数值和状态，不采用营销式大卡片或装饰性渐变。
- 主色：深蓝灰结构色、工艺蓝操作色、青绿有效结果、琥珀待保存/覆盖状态。
- 密度：默认“标准”，可通过 Tweaks 切换舒展或紧凑。
- 形状：4—6px 小圆角、细边框、轻阴影。
- 字体：`Segoe UI Variable Text`、`Microsoft YaHei UI` 等本地桌面字体。

## 页面映射

| 截图 | 需求重点 |
|---|---|
| `01-项目库.png` | 项目搜索、新建、从收资文件新建、版本与更新时间概览 |
| `02-需求汇总.png` | 项目/楼层可累加指标、非累加参数分布、气象条件 |
| `03-计算总表.png` | 参数为行、房间为列、固定参数列、分组折叠、输入/结果区分 |
| `04-房间编辑侧栏.png` | 总表只读、房间分组编辑、应用并重算、复制新建 |
| `05-收资导入预览.png` | 全量覆盖、新增/覆盖/冲突预览、整批导入提醒 |

## 生产实现必须保留

- 顶部清楚显示当前项目、方案版本、计算规则版本和未保存状态。
- 项目库不显示方案级“保存/导出”操作。
- 计算总表是主要工作视图，汇总页是概览入口。
- 房间输入与公式结果视觉上有稳定区分，但页面不展示公式。
- 编辑侧栏打开时，背景总表仍提供上下文但不可误操作。
- 导入确认必须明确说明空白单元格也会清空现有值。

## 可以调整

- 精确字号、色值和阴影。
- 分组默认展开状态。
- 汇总卡片数量和排序。
- 总表使用的具体虚拟表格组件。
- “应用”和“保存方案”的按钮文案，但不得弱化未保存提示。
