# 洁净区暖通计算工作台需求澄清记录

本文记录需求访谈中已经确认的产品约束；未确认事项不写作正式需求。

## 产品范围

- V1 是单人操作的计算工作台，不包含多人协作、审核或审批流程。
- 使用一个共享入口账户和口令限制访问。
- 项目长期保存在共享项目库中，可后续继续编辑和复用。
- V1 只输出设备选型所需的设计参数，不维护厂家或产品型号库，也不自动推荐具体设备。
- 具体厂家、型号及最终设备方案由外部选型机构完成。

## 部署与访问

- 系统部署为内网 Web 应用，不开放公网访问。
- 项目、房间和方案版本统一保存在内网服务器中。
- 设计人员通过局域网中的浏览器访问计算工作台。
- V1 不设置个人用户、操作者姓名或个人审计记录。
- 方案编辑锁只关联浏览器会话；锁冲突时不显示个人身份。
- 共享入口账户和口令保存在服务器配置中，由部署维护人员修改。
- V1 不提供账户管理页面、个人密码修改或忘记密码流程。
- V1 暂不包含项目库自动备份及用户侧恢复功能。
- V1 只面向桌面浏览器，支持内网电脑上的最新版 Edge 和 Chrome。
- 页面按至少 1366 像素宽度设计，不适配手机和平板。
- 不使用 Docker 部署。
- 应用需要支持在 Linux 或 Windows 内网服务器上原生运行。
- 项目库使用应用内置的 SQLite 数据库，不依赖独立数据库服务。
- 应用采用 Python Flask 单体架构，不拆分独立前端应用或微服务。
- 页面采用 Jinja 模板、HTMX 和少量原生 JavaScript 实现交互。
- Excel 导入导出由 Python `openpyxl` 统一处理。
- 计算规则实现为独立且可版本化的 Python 模块。

## 核心目标

- 版本控制：用户可以主动保存和读取项目的方案版本。
- 方案简化：减少 Excel 式的分散录入、查表和固定列操作。
- 公式隐藏：普通页面不展示且不允许编辑公式，但不要求对技术人员保密或加密。

## 方案版本

- 系统在项目内自动生成 `V1、V2…` 形式的版本编号。
- 用户保存版本时可以填写版本名称和备注。
- 系统自动记录保存时间；派生版本还应记录来源版本。
- 方案版本保存当时的项目级条件、全部房间参数和计算结果。
- 用户可以单独删除方案版本，删除前必须确认。
- 当前正在编辑的方案版本不能直接删除，用户必须先切换到其他版本。
- V1 只要求用户主动保存和读取，不包含自动编辑历史、版本差异比较、审批或分支管理。
- 读取已有方案版本后默认进入可编辑状态，允许继续修改原版本。
- 用户需要保留原版本时，可以 Fork 为派生版本后再修改。
- 派生版本保留其来源版本关系，但 V1 不要求版本差异比较。
- 方案版本仅在用户明确点击保存时更新，不自动覆盖。
- 存在未保存修改时，页面应明确提示，并在离开前提醒用户。
- V1 初始交互中，房间侧边面板使用“应用”更新当前工作副本并重新计算；页面顶部“保存方案”才将全部修改写入方案版本。
- 上述保存交互允许在后续原型验证中调整，但必须保持未保存状态可见。
- 同一方案版本不允许多个会话并发修改，也不允许后保存者静默覆盖先保存的数据。
- V1 不提供实时协同编辑。
- 第一个进入编辑状态的会话获得方案编辑锁，其他会话只能查看或 Fork。
- 保存、取消或正常退出时释放编辑锁；连续 30 分钟无活动时自动释放异常遗留的编辑锁。

## 项目与房间

- 项目编号、项目名称和城市为项目必填信息。
- 项目编号在项目库内唯一，用于搜索和标识结果文件。
- 客户/建设单位、项目地址和项目备注为可选信息。
- V1 支持直接永久删除项目，不设置回收站。
- 删除前必须明确确认将同时删除项目下全部房间和方案版本；删除后系统不提供恢复。
- 一个项目包含一个或多个房间。
- 同一项目内所有房间共用一套室外设计条件。
- 室外设计条件由城市标准气象值带出，并允许项目级人工覆盖。
- 人工覆盖气象参数时，系统同时保留标准值和当前采用值，并标记“已覆盖”。
- 结果文件需要注明项目存在人工覆盖的室外气象参数。
- 人工覆盖支持单项恢复或全部恢复标准值；恢复后清除覆盖标记并重新计算当前方案。
- V1 内置权威模板中的全国室外气象参数库，数据只读，不提供维护页面。
- 气象参数库的新增或更新随新的计算规则版本发布。
- 房间编号在项目内唯一，也是重复导入时匹配房间的依据。
- 房间可以设置楼层和区域，用于筛选、批量操作及收资整理；不通过解析房间编号推断楼层。
- V1 单个项目最多包含 200 个房间。
- V1 不维护独立的房间参数预设库，使用已有典型房间作为复制来源。
- 新增房间时支持从一个已有房间复制新建。
- 复制新建时复制楼层、区域及全部可编辑计算参数，不复制房间编号、房间名称和计算结果。
- 新房间必须填写新的唯一房间编号和名称，并由系统重新计算结果。
- 复制新建房间只允许在同一项目内使用；跨项目复用通过收资文件导出和导入完成。

## 计算规则

- 当前暖通 Excel 是参数含义、单位、默认值和有效公式的权威模板。
- 系统必须等价实现模板中的有效公式，但不复刻工作表行列、颜色和操作顺序。
- 模板中的断链公式列为待补充公式，确认前不推测、不形成正式结果。
- 系统内部计算保留完整精度，除非权威模板公式明确使用 `ROUNDUP`、`FLOOR` 等取整函数。
- 页面和结果文件按权威模板的显示格式呈现小数位，但显示格式不改变中间计算值。
- 权威模板中每个有效公式都必须建立系统与 Excel 的对照验收用例。
- 对照验收使用相同输入；内部值按完整精度比对，页面显示值按模板格式比对。
- 待补充公式不纳入当前计算验收，确认并实现后再加入验收范围。
- 待补充公式对应的结果显示明确状态，不得显示为零值或空白。
- 汇总展示排除待补充公式的结果，并提示当前结果不完整。
- 结果文件同样标注待补充公式和结果不完整状态。
- V1 范围内所有待补充公式必须在正式上线前闭环；未闭环时计算功能不得判定验收完成。
- 系统计算规则具有独立版本标识，已发布的计算规则版本不可变。
- 每个方案版本绑定一个计算规则版本，旧方案不会因后续补充或调整公式而自动改变结果。
- 用户需要升级计算规则时，通过 Fork 创建派生版本，并使用最新规则重新计算。

## 数据交换

- 收资文件只传递可编辑的项目和房间输入参数，可导出、填写并重新导入。
- 收资文件统一为 `.xlsx`，包含填写说明、项目信息与室外条件、房间参数等工作表。
- 房间参数工作表采用“参数纵向、房间横向”，一个房间占一列，与网页计算总表方向一致。
- 收资文件的参数名称和排列应与计算总表中的对应参数保持一致，并包含单位、必填标识和可选值说明。
- 收资文件不包含计算公式。
- 收资文件支持导出项目内全部房间或用户选中的房间。
- 结果文件只交付表三计算参数及设备选型参考，不支持反向导入。
- 交换文件结构可以与权威模板不同，但不能改变字段含义和计算语义。
- 导入时按房间编号区分新增、更新和冲突，不静默覆盖。
- 对房间编号匹配的已有房间执行全量覆盖导入：文件中的全部可编辑参数替换系统现有值，空白单元格也会清空原值。
- 向已有项目导入收资文件时，只处理房间参数，不覆盖项目名称、城市、室外设计条件或其他项目级信息。
- 项目库支持从收资文件直接新建项目，并读取项目编号、名称、城市、室外设计条件和全部房间。
- 从收资文件新建项目时，如项目编号已存在则阻止创建并提示修改。
- 正式导入前必须明确展示将新增、覆盖和冲突的房间，并由用户确认。
- 收资文件任一房间存在导入错误时，整批数据均不导入，项目保持原状。
- 错误提示必须定位到工作表、参数行和房间列，并说明错误原因。
- 导入校验分为错误和警告两级。
- 必填缺失、房间编号重复、数据类型错误、非法选项及无法计算等错误会阻止整批导入。
- V1 不设置工程参数合理范围或阈值报警，也不因数值偏离常见范围阻止导入。
- 警告级校验能力可以保留，但 V1 暂不配置工程阈值规则。
- 结果文件统一为 `.xlsx`。
- 结果文件包含“项目/楼层需求汇总”和“全部房间计算总表”两张工作表。
- 结果文件导出结果值、单位、项目名称、方案版本和导出时间，不包含计算公式。
- PDF 导出不属于 V1。

## 结果展示

- V1 提供汇总展示页，作为项目结果的概览入口。
- 汇总展示页至少包含项目总需求、楼层需求和关键数量表。
- 复杂图表、趋势分析和高级分析不属于 V1。
- 面积、冷热负荷、送风量、新风量、回风量、各类排风量、设备数量、功率及水量等可累加指标按项目和楼层求和。
- 温湿度、洁净等级、压差、换气次数、风速及比例等非累加参数不求和，只展示范围或分类分布。
- V1 以一张类似权威模板表三的计算总表作为主要结果视图。
- 计算总表汇总项目内全部房间的输入参数、计算结果和设备选型参考。
- V1 不单独开发房间结果详情页；后续可根据使用反馈调整。
- 计算总表只读，不直接修改输入参数。
- 用户选择房间后，通过侧边编辑面板修改该房间参数；应用后重新计算并刷新总表。
- 房间侧边编辑面板只展示可编辑字段，不展示公式结果。
- 编辑字段按基本信息、室内条件、洁净与排风、负荷、设备与构造参数分组。
- 基本信息默认展开，高级工程参数默认折叠，必填项优先展示。
- 计算总表保持权威模板表三的方向：参数纵向排列，房间横向排列。
- 页面固定参数列，并支持按楼层和区域筛选当前显示的房间。
- 页面可以只呈现当前筛选的房间，结果文件能够导出项目内全部房间。
- 总表保留权威模板表三的全部参数，并按基础参数、洁净与正压、冷热负荷、风量、FFU、干盘管、回风夹道及百叶等计算阶段分组。
- 参数组支持展开和折叠，并支持按参数名称搜索。
- 默认展开常用参数组，其余参数组默认折叠。
