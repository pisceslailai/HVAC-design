# 洁净区暖通计算

本上下文描述暖通设计人员如何按项目和房间收集参数、完成洁净区暖通计算，并形成设备选型参考结果。

## Language

**计算工作台**:
面向暖通设计人员、以单人操作方式完成暖通参数录入和计算的工作空间。
_Avoid_: 网站、网页、多人项目管理系统

**项目**:
一次暖通计算任务的边界，包含项目级条件和一个或多个房间。

**项目编号**:
在项目库内唯一、用于识别项目及标记交换文件的业务编号。

**项目库**:
计算工作台中长期保存全部项目、供后续继续编辑和复用的共享集合。

**房间**:
项目中独立录入参数并产生计算结果的最小计算对象。

**房间编号**:
在同一项目内唯一、用于识别房间及匹配重复导入数据的业务编号。

**楼层**:
房间在项目建筑中的楼层归属，用于组织、筛选和批量操作。

**区域**:
项目内用于组织相关房间的可选业务分组。

**复制新建房间**:
以一个已有房间的参数作为初值创建新房间的操作。
_Avoid_: 就近复制

**室外设计条件**:
项目所在地的一组室外气象计算参数，由项目内所有房间共用。

**标准气象值**:
根据项目城市从气象参数库取得的室外设计条件原始值。

**项目覆盖值**:
设计人员针对当前项目人工修订、用于替代标准气象值的室外设计条件值。

**权威模板**:
定义系统参数含义、单位、默认值和计算公式，但不限制页面组织形式的现有暖通 Excel 模板。
_Avoid_: 参考样例、非约束性模板

**计算语义**:
由权威模板定义的参数含义、单位、默认值、计算公式及其适用关系。

**交互组织**:
设计人员录入、查看和复核参数的分组、顺序与页面流程。
_Avoid_: Excel 行列布局

**系统计算规则**:
对权威模板中有效公式和计算关系的等价实现。

**计算规则版本**:
一套确定且不可变的系统计算规则标识，用于固定方案版本的计算依据。

**计算总表**:
在一个项目中集中展示全部房间输入参数、计算结果和设备选型参考的主要结果视图。
_Avoid_: 单房间结果页、分散结果卡片

**汇总展示**:
按项目整体或楼层聚合房间计算结果，用于查看总体需求和分组需求的结果视图。
_Avoid_: 计算总表、单房间结果

**可累加指标**:
能够在房间、楼层和项目层级直接求和并保持工程意义的需求量或设备数量。

**非累加参数**:
不能直接求和、只能展示范围或分类分布的房间设计条件和比率。

**方案版本**:
由设计人员主动保存、可在之后读取并继续编辑的项目参数与计算结果状态。
_Avoid_: 不可变审计快照、自动编辑历史、审批版本

**版本编号**:
由系统按项目自动生成、用于区分方案版本的顺序标识。

**派生版本**:
从已有方案版本复制形成、并保留来源关系的新方案版本。
_Aliases_: Fork

**公式隐藏**:
计算工作台的普通交互不展示也不允许编辑系统计算规则，但不承担技术保密要求。
_Avoid_: 商业机密保护、加密公式

**待补充公式**:
权威模板中已经断链或缺失、尚不能形成正式计算结果的公式。

**收资文件**:
用于导出、线下填写并重新导入项目和房间可编辑参数的交换文件。
_Avoid_: 结果文件、系统存档

**全量覆盖导入**:
以收资文件中的完整房间参数替换系统内同编号房间全部可编辑参数的导入方式。

**结果文件**:
用于交付表三计算参数及设备选型参考、但不能反向导入的输出文件。
_Avoid_: 收资文件、计算输入

**选型机构**:
根据系统输出的设计需求参数完成具体设备厂家和型号选择的外部协作方。

**设计人员**:
进入计算工作台并执行收资、计算和结果复核的人。

**入口账户**:
所有设计人员共用、仅用于限制计算工作台访问的单一账户和口令。
_Avoid_: 个人账号、用户角色、审批账户

## Relationships

- 一个 **项目** 包含一个或多个 **房间**
- 一个 **项目** 在 **项目库** 中具有唯一的 **项目编号**
- 一个 **房间** 只属于一个 **项目**
- 一个 **房间** 在所属项目内具有唯一的 **房间编号**
- 一个 **房间** 可以归属于一个 **楼层** 和一个 **区域**
- **收资文件** 使用 **房间编号** 将数据匹配到已有房间
- **全量覆盖导入** 对匹配到的已有房间替换全部可编辑参数，包括以空白清空原值
- **复制新建房间** 从已有 **房间** 创建新的独立房间
- 一个 **项目** 使用一套 **室外设计条件**
- **室外设计条件** 默认采用 **标准气象值**，并可被 **项目覆盖值** 替代
- **权威模板** 决定 **系统计算规则** 的初始数据、默认值和计算关系
- **系统计算规则** 保持 **计算语义** 一致，但 **交互组织** 可以重新设计
- 一个 **方案版本** 使用且只使用一个 **计算规则版本**
- **方案版本** 继续编辑时沿用原计算规则版本，升级规则时通过 **派生版本** 重新计算
- **待补充公式** 在确认前不属于可执行的 **系统计算规则**
- **系统计算规则** 根据项目和房间参数产生计算结果
- **收资文件** 在计算工作台与外部收资过程之间双向传递可编辑参数
- **系统计算规则** 产生的计算结果可以形成 **结果文件**
- **结果文件** 向 **选型机构** 提供设备选型依据
- 一个 **项目** 通过一张 **计算总表** 汇总全部房间的计算参数
- **汇总展示** 使用 **计算总表** 中可聚合的房间结果形成项目和楼层需求
- **汇总展示** 对 **可累加指标** 求和，对 **非累加参数** 展示范围或分类分布
- 一个 **项目** 可以保存一个或多个 **方案版本**
- 一个 **方案版本** 在所属项目中具有唯一的 **版本编号**
- **设计人员** 可以保存和读取 **方案版本**
- **方案版本** 默认可以继续编辑
- **设计人员** 可以从一个 **方案版本** 创建 **派生版本**
- **公式隐藏** 使 **设计人员** 无法查看或修改 **系统计算规则**
- **设计人员** 使用 **入口账户** 进入 **计算工作台**
- **入口账户** 访问同一个 **项目库**
- **项目库** 包含零个或多个 **项目**

## Example dialogue

> **开发人员：** “设计人员进入计算工作台后，是一次编辑一个房间吗？”
> **领域人员：** “不是。一次处理一个项目，项目中可以逐个录入多个房间，再统一查看计算结果。”

## Flagged ambiguities

- “登录权限”曾可能指个人账号体系，现已明确为所有设计人员共用的 **入口账户**。
- “Excel 模板”曾被描述为参考材料，现已明确当前文件是定义数据、默认值和有效公式的 **权威模板**。
- “以 Excel 为准”仅约束 **计算语义**，不要求复刻工作表、单元格、颜色或操作顺序。
- 模板中的 `#REF!` 断链项已确定为 **待补充公式**，确认前不得推测计算。
- “版本控制”已明确为用户主动保存和读取 **方案版本**，V1 不包含自动编辑历史、差异比较或审批流程。
- “公式隐藏”仅指页面不展示、不可编辑，不要求防止技术人员检查程序中的计算逻辑。
